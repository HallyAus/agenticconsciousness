```javascript
/** @type {import('next-sitemap').IConfig} */
module.exports = {
  siteUrl: process.env.SITE_URL || 'https://agenticconsciousness.com.au',
  generateRobotsTxt: true,
  generateIndexSitemap: false,
  changefreq: 'weekly',
  priority: 0.7,
  sitemapSize: 5000,
  robotsTxtOptions: {
    additionalSitemaps: [],
    policies: [
      { userAgent: '*', allow: '/' },
      { userAgent: '*', disallow: '/api/' },

    // Set homepage as highest priority
    if (path === '/') {
      return { loc: path, changefreq: 'weekly', priority: 1.0, lastmod: new Date().toISOString() };
    }
    return { loc: path, changefreq: config.changefreq, priority: config.priority, lastmod: new Date().toISOString() };
  },
};
```

## PHASE 3: TOKEN USAGE & API COST OPTIMISATION

The chatbot uses the Anthropic API. Optimise to prevent runaway costs.

### 3.1 — Rate Limiting on /api/chat

Update `src/app/api/chat/route.ts` to add IP-based rate limiting:

- Maximum 10 requests per minute per IP address
- Maximum 50 requests per hour per IP address
- Return 429 Too Many Requests with a friendly message when exceeded
- Log rate limit hits for monitoring

If using Upstash Redis:
```typescript
import { Ratelimit } from '@upstash/ratelimit';
import { Redis } from '@upstash/redis';

const ratelimit = new Ratelimit({
  redis: Redis.fromEnv(),
  limiter: Ratelimit.slidingWindow(10, '1 m'),
  analytics: true,
});
```

If NOT using Upstash (simpler in-memory approach):
Create a simple in-memory rate limiter using a Map with IP keys and timestamp arrays. Reset entries older than 1 minute. This works for single-instance deployments.

### 3.2 — Conversation History Trimming

Already implemented (last 20 messages), but additionally:
- Count estimated tokens before sending (rough: 4 chars = 1 token)
- If estimated input tokens exceed 3000, trim older messages more aggressively (keep last 10)
- Add a "conversation too long" message after 30 exchanges suggesting the user book a real consultation

### 3.3 — System Prompt Token Efficiency

The system prompt is ~250 tokens. This is fine. Do NOT inline it on every request — store it as a constant. Already done in the build spec.

### 3.4 — Response Token Cap

Already set to `max_tokens: 1024`. This is correct for concise chatbot responses. Do not increase.

### 3.5 — Caching Common Responses

Create a simple response cache for common questions:
- "What do you do?" / "What services do you offer?" / "How much does it cost?" / "Where are you based?"
- Store 5-10 cached responses in a Map
- Match incoming messages against cached patterns (case-insensitive keyword matching)
- Return cached response instantly without an API call
- This saves significant token costs for the most common visitor questions

### 3.6 — Usage Monitoring

Add structured logging to the chat route:
```typescript
console.log(JSON.stringify({
  event: 'chat_request',
  ip: req.headers.get('x-forwarded-for') || 'unknown',
  input_tokens: response.usage.input_tokens,
  output_tokens: response.usage.output_tokens,
  model: 'claude-sonnet-4-20250514',
  timestamp: new Date().toISOString(),
}));
```

This lets you track daily token spend and identify abuse patterns.

## PHASE 5: PERFORMANCE AUDIT

### 5.1 — Bundle Size Analysis

Add to `next.config.ts`:
```typescript

  enabled: process.env.ANALYZE === 'true',

```

Run: `ANALYZE=true npm run build`

Review the report. Target:
- First Load JS < 100KB
- Largest component < 30KB
- No unnecessary dependencies in the client bundle
- `@anthropic-ai/sdk` must NOT appear in client bundles (server-only)

### 5.2 — Font Loading

The site uses Google Fonts (Be Vietnam Pro + Space Mono). Optimise:
- Use `next/font/google` instead of the CSS import for automatic font optimisation
- This eliminates the render-blocking Google Fonts request

Update `src/app/layout.tsx`:
```typescript
import { Be_Vietnam_Pro, Space_Mono } from 'next/font/google';

const beVietnamPro = Be_Vietnam_Pro({
  subsets: ['latin'],
  weight: ['300', '400', '500', '700', '900'],
  variable: '--font-display',
  display: 'swap',
});

const spaceMono = Space_Mono({
  subsets: ['latin'],
  weight: ['400', '700'],
  variable: '--font-mono',
  display: 'swap',
});
```

Apply the CSS variables to `<body>`.

### 5.3 — Core Web Vitals Targets

- **LCP** (Largest Contentful Paint) < 2.5s — the hero h1 should paint fast. No blocking resources.
- **FID** (First Input Delay) < 100ms — the chatbot JS should be lazy-loaded.
- **CLS** (Cumulative Layout Shift) < 0.1 — fonts use `display: swap`, no layout-shifting images.

### 5.4 — Lazy Load the Chatbot

The chatbot component should be dynamically imported:
```typescript
import dynamic from 'next/dynamic';
const Chatbot = dynamic(() => import('@/components/Chatbot'), { ssr: false });
```

This keeps the chatbot JS out of the initial page load.

### 5.5 — Image Optimisation

If any images are added later (case study screenshots, team photos):
- Use `next/image` with automatic optimisation
- Set explicit `width` and `height` to prevent CLS
- Use `priority` on above-the-fold images only
